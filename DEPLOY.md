# Размещение бота на бесплатном хостинге (Railway.app)

Чтобы бот работал круглосуточно без вашего компьютера, его нужно "переселить" на сервер.
Самый простой бесплатный вариант для новичка — **Railway.app**.

Общая логика: код заливается на GitHub (это как облачное хранилище для кода) →
Railway подключается к этому GitHub-репозиторию и сам запускает бота на сервере.

## Шаг 1. Завести аккаунт на GitHub

1. Зайти на https://github.com
2. Зарегистрироваться (email + пароль).

## Шаг 2. Установить GitHub Desktop (программа для загрузки кода без командной строки)

1. Скачать с https://desktop.github.com/
2. Установить, при запуске войти под своим GitHub-аккаунтом.

## Шаг 3. Создать репозиторий и загрузить туда код

1. В GitHub Desktop: File → New Repository.
2. Name: например `tg-summary-bot`.
3. Local Path: выбрать папку, где лежит ваш проект (`bot.py`, `requirements.txt` и т.д.) — либо указать путь к уже существующей папке `tg_summary_bot`.
4. Нажать Create Repository.
5. Убедитесь, что в папке проекта лежат новые файлы, которые я добавил (скачайте их ниже): `Procfile` и `.gitignore`.
6. **Важно:** файл `.env` со своими токенами НЕ должен загружаться на GitHub — это публичное место, и токены могут украсть. Файл `.gitignore` как раз следит, чтобы `.env` не попал в загрузку (он его игнорирует автоматически).
7. В GitHub Desktop внизу слева написать любой комментарий (например "первая загрузка") и нажать **Commit to main**.
8. Сверху нажать **Publish repository**. Важно: убрать галочку "Keep this code private" можно оставить как есть — приватный репозиторий тоже подходит.

## Шаг 4. Зарегистрироваться на Railway и подключить тариф Hobby

Важное уточнение по актуальным ценам: в 2026 году у Railway нет постоянного бесплатного тарифа для таких ботов — есть только разовый пробный кредит на $5 при регистрации. Для стабильной круглосуточной работы бота нужен тариф **Hobby** — он стоит $5/мес, и в эту сумму уже включён лимит потребляемых ресурсов. Такой лёгкий бот, как наш, скорее всего не выйдет за пределы включённого лимита, то есть по факту вы будете платить примерно эти $5/мес и не больше (но это не гарантия — смотрите свои расходы в разделе Usage в личном кабинете).

1. Зайти на https://railway.app
2. Нажать Login → войти через GitHub-аккаунт (тот же, что использовали выше). Это удобно — Railway сразу увидит ваши репозитории.
3. Перейти в Account Settings → Billing.
4. Привязать банковскую карту.
5. Выбрать тариф **Hobby** ($5/мес).
6. После этого можно переходить к созданию проекта — сервисы на Hobby-плане не "засыпают" из-за неактивности, что критично для polling-бота.

## Шаг 5. Создать проект на Railway

1. На главной странице Railway: **New Project**.
2. Выбрать **Deploy from GitHub repo**.
3. Найти и выбрать ваш репозиторий `tg-summary-bot`.
4. Railway начнёт разворачивать проект автоматически — он увидит `requirements.txt` и `Procfile` и поймёт, как установить зависимости и запустить бота.

## Шаг 6. Указать секретные токены (переменные окружения)

Так как `.env` мы не загружали, токены нужно вписать прямо в настройках Railway:

1. Открыть созданный проект → вкладка **Variables**.
2. Добавить две переменные (кнопка "New Variable"):
   - `BOT_TOKEN` = ваш токен от BotFather
   - `ANTHROPIC_API_KEY` = ваш ключ Anthropic
3. Сохранить — Railway автоматически перезапустит бота с этими переменными.

## Шаг 7. Проверить, что бот запустился

1. Во вкладке **Deployments** → открыть последний деплой → вкладка **Logs**.
2. Там должна появиться строка "Бот запущен" (как раньше в терминале на компьютере).
3. Написать боту в Telegram `/summary` — проверить, что отвечает.

## Важный нюанс: хранение истории сообщений

Бот хранит историю в файле `chat_history.db` (SQLite) прямо на диске сервера. На бесплатном плане Railway при каждом новом деплое (например, если вы обновите код) файловая система может пересоздаваться, и накопленная история сообщений **сотрётся**.

Если для вас это важно (не терять историю между обновлениями кода), два варианта:
- **Простой:** просто не пересобирайте проект без необходимости — пока бот работает без изменений, база данных сохраняется.
- **Надёжный:** подключить Railway Volume (постоянное хранилище) — платная функция, но недорогая, либо перейти на внешнюю БД (например, бесплатный PostgreSQL от Railway или Supabase). Если дойдёте до этого — могу расписать отдельно, это чуть сложнее для новичка.

## Частые проблемы

- **В логах ошибка про отсутствующие переменные (BOT_TOKEN)** → проверьте, что вписали Variables на Railway (Шаг 6), и что в коде используется `os.getenv("BOT_TOKEN")`, а не захардкоженный токен.
- **Бот не отвечает после деплоя** → откройте Logs и посмотрите на ошибку — обычно текст ошибки сразу подсказывает причину.
- **Не хочется привязывать карту** → используйте Render.com вместо Railway.
